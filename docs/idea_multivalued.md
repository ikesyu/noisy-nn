# 多価データからのノイズ場自己分割 — 同じ $x$ に複数の $y$ があるとき、場はいくつに分かれるべきか

## 1. 本稿の目的

`docs/idea_consolidation.md`（以下「コンソリデーション稿」）は、ノイズ場の支持を縮小する手続き（圧縮）と、空いた領域へ複数の関数を逐次格納する手続き（容量）を扱った。そこでは**タスク境界が外部から与えられる** — どのデータがどのタスクに属するか、いくつのタスクがあるかは既知で、id 1..K が振られ、タスク記述子（場マスク + 読み出し + 出力バイアス）を復元して関数を切り替える — ことを前提にしていた。この前提は、コンソリデーション稿 §14.5（SupSup との比較）で「タスク ID 推定の欠如」として今後の課題に明記されている。

本稿はその欠落を、**データ自身に境界を決めさせる**方向から埋める。扱う状況は次のとおりである：

> 学習データに、同じ入力 $x$ に対して複数の出力 $y$ が現れる（多価関係）。それらを一つのノイズ場に収めることが原理的にできない。このとき、ネットワークが自ら多価性を検出し、ノイズ場を分割して新しい id を割り当て、学習を継続する枠組み。

ここで得たいのは単なる混合モデルではなく、**分割の判定・分岐の格納・資源会計のすべてが、コンソリデーション稿で確立した機構（動員ダイヤル $\rho$、消滅経路、閉ループ、forward 統計、ソフト共有）の上で完結すること**である。

## 2. 問題の正確な形：なぜ一つの場に収まらないか

同じ $x$ に $y_1, y_2$ が対応するとき、二乗誤差の最小化は必然的に条件付き平均へ収束する：

$$
\arg\min_{f}\ \mathbb E_{x,y}\big[(f(x)-y)^2\big] = \mathbb E[y\mid x],
$$

残差は条件付き分散 $\mathrm{Var}[y\mid x]$ に下げ止まり、ユニットを増やしても epoch を増やしても減らない。$y=\pm g(x)$ が等頻度なら $\mathbb E[y|x]=0$ で、出力は**両分岐のどちらでもない値に潰れる**。

NNN 特有の事情がここに重なる。ネットワークは元々確率的なので「出力がばらつくこと」自体は異常ではない。しかし交差ノイズ由来のゆらぎは $T$ サンプルの平均を経てほぼ単峰（中心極限的）であり、**分岐間の多峰性を表現できない**。したがって：

> **内在ノイズは「分岐内のばらつき」を担えるが、「分岐間の多峰性」は担えない。** 前者は $\sigma$ の大きさの問題、後者は場の個数の問題である。この役割分担が、場を分ける必然性そのものである。

逆に言えば、残差が広いだけ（単峰）なら分割してはならない — それは一つの場が $\sigma$ で表現すべき範囲である。**「広い」と「多峰」の区別が、分割判定の核心**になる。

## 3. 分割の引き金：三つの交絡を切り分ける

損失が下がらない原因は三つあり、混同すると誤った処置に至る。コンソリデーション稿 §12.9.8 で容量とタスク難易度の交絡に実際に躓き、対照実験で切り分けた経験がそのまま活きる。

| 原因 | 見分け方（すべて forward 統計） | 正しい処置 |
|---|---|---|
| 容量不足 | 空きユニットの動員で損失が下がる | 通常の逐次格納（動員を増やす） |
| 学習可能性の限界 | 全ネットワークの単独学習でも下がらない（§12.9.12 の対照） | 基底の解像度（マルチスケール初期化など） |
| **データの多価性** | 空きがあり動員しても下がらない、かつ**アンサンブルのパス数 $P$ を増やしても残差が下げ止まる**、かつ**残差分布が多峰** | **場の分割**（本稿） |

**判定の二段構え。**

1. **既約性**：予測を $P$ パスのアンサンブル平均 $\bar y_P(x)$ にしても $\lVert \bar y_P(x)-y\rVert$ が下げ止まる。内在ノイズ由来なら $1/P$ で減るので、$P$ 依存性の有無が両者を分ける。
2. **多峰性**：残差分布 $\{y_n - \bar y(x_n)\}$ が、単に広いのでなく多峰である。

多峰性検出は NNN 的に自然に書ける。交差活性は分布フリーな密度推定 $(\mathrm{xor}_2-\mathrm{xor}_1)/(2h)$ を内蔵しているので、**残差に対してしきい値を掃引する交差ユニットを一つ置けば、残差密度の形が forward 統計だけで読める**（外部のヒストグラムも EM も要らない）。

**最も自然な引き金は、既にログしている `holds` の異常である。** コンソリデーション稿 §12.5 は「容量枯渇は在庫（$S_k$）でなく動学の破綻（holds）に最初に現れる」と結論したが、多価性はそれと区別可能な形で現れる：**空きユニットが潤沢にあるのに holds が飽和する**（容量枯渇は空きゼロで飽和する）。既存の閉ループのログがそのまま検出器になる。

## 4. 分割の仕方：分岐は同じ部分空間に住む

多価関係の分岐は、たいてい**入力構造の大半を共有し、低次元的にだけ違う**。極端な例が $y=\pm g(x)$ で、これは読み出し係数の符号反転だけで表せる — コンソリデーション稿 §12.9.4 で実測した $\sin$ と $-\sin$ の関係（読み出しコサイン類似度 $-1.00$、自前 L2 ユニット 0 個）そのものである。

したがって分割は hard（互いに素な領域）ではなく、**ソフト・コンソリデーションが自然な既定**になる。段階的に降りる：

1. **読み出し共有（案2/3）**：既存場を凍結語彙とし、新 id は読み出し列だけを学習する。分岐が既存の張る空間内なら自前ユニットはほぼ 0。
2. **二層共有（案A）**：分岐が入力構造まで違う場合、過去 L1 のバンプ基底を新 L2 が読む。
3. **新規領域**：どちらも足りない場合のみ。

コンソリデーション稿 §12.9.13 の結論「容量＝張れる部分空間の数 × 各部分空間の再利用回数」は、ここでは**「分岐の追加はほぼ無料であるべき」という定量的予測**になる。

## 5. 割当：E-step は既存の計算そのもの

どのデータ点がどの id に属するかは鶏と卵であり、EM の形をとる。NNN では E-step が既存演算と一致する：

- **E-step**：各データ点について全 id の記述子を復元して forward 評価し、残差最小の場へ割り当てる（またはソフト責任度 $r_{ni} \propto \exp(-(y_n-\bar y_i(x_n))^2/2s^2)$）。これはコンソリデーション稿 §12.8.2 以降で毎回作っている**クロス評価行列と同一の演算**であり、backward を一切使わない。
- **M-step**：各場が自分の担当部分集合で cov_jac 学習（forward-only）。
- **新規 id の生成**：どの既存場も説明できない点（全 id で残差 > 閾値）が集積したら分割。SupSup（§14.5）が出力エントロピーで行う不確実性検出の**残差版**にあたる。

## 6. 分割しすぎの抑制：ユニットが MDL コストになる

EM 系の常識的な危険は「分割するほど残差が減る」ことである。本構想は**自然な正則化**を持つ：新しい場を切るコストは**実ユニット数**であり、それは稼働 PE・消費電力に一対一対応する実資源である（§8, §12.9.13.2）。分割判定を

$$
\Delta(\text{残差}) \ >\ \lambda \cdot \Delta(\text{占有ユニット数})
$$

と書けば、モデル選択が資源会計そのものになる。しかも再利用曲線により、**真の分岐（同じ部分空間）は数ユニットしか要求せず、偽の分割（ノイズへの過適合）は新しい部分空間を要求して高くつく** — コストが自動的に真偽を弁別する方向へ働く。

## 7. 推論の意味論：id が潜在変数になる

分割後、テスト時に $x$ だけ与えられたときの出力には三つの意味論がある。

1. **文脈が与えられる**：記述子で選ぶ（コンソリデーション稿の現行動作）。
2. **集合値出力**：多価関係の正しい意味論は「全分岐が正解」なので、全 id の予測を返す。
3. **サンプリング**：id を潜在変数とみなし、$p(y|x)$ のモードを選ぶ。

3 の見方では、**場の集合が共有基板上の混合モデルになり、各場が条件付き分布の一つのモードを担い、内在ノイズがモード内のばらつきを担う**。通常のネットワークが混合密度ネットワークや潜在変数モデルという専用機構を要するのに対し、ここではモードが物理的な場として分離される。

## 8. 検証計画

**PoC-M1（本稿の最初の実験）**：$x\in[-2\pi,2\pi]$ に対し $y=\sin x$ と $y=-\sin x$ を各 50% で提示する多価データ。予測：

- (a) 単一場では出力が $\bar y \approx 0$ に潰れ、残差が下げ止まる（ユニットを増やしても $P$ を増やしても改善しない）。
- (b) 既約性 + 多峰性の判定が発火する。
- (c) 分割後、第二分岐は**自前ユニットほぼ 0・読み出し符号反転**で格納される（§12.9.4 の実測がそのまま再現するはず）。
- (d) 分割後の各場の残差が、単一場の下げ止まり値から大きく下がる。

**PoC-M2**：部分的多価性（$x>0$ の領域だけ分岐する）。場が**部分的に重なる**構成になるか、重なりの境界が多価領域の境界と一致するかを見る。案5（重なり限定リハーサル）の適用領域。

**PoC-M3**：分岐数が未知（3 分岐以上、非等頻度）。分割の停止則と、稀な分岐が条件付き平均に埋もれずに検出されるかの限界を測る。

**PoC-M4**：検出（M1/M2）と分割ループ（M3）の**オンライン統合**。多価かどうかを知らされないタスク列を一気通貫で処理し、品質ゲート → 三者鑑別（容量・既約性・条件付き広がり）→ 分割 or 棄却、の自動判定を検証する。学習不能な単価値タスクを陰性対照に含める。

## 9. 予見される難所

1. **割当の初期不安定性**：EM の局所解。初期は分割を保留し、証拠が積み上がってから切る保守性が要る。
2. **部分的多価性の扱い**：定義域の一部だけで分岐する場合、場は部分的に重なるべきで、単純な排他割当は誤る。
3. **非対称な分岐頻度**：稀な分岐が条件付き平均に埋もれて検出されない可能性。
4. **多峰性検出の感度**：分岐が近接している（$|y_1-y_2|$ が内在ノイズ幅と同程度）場合、多峰性と単なる広がりの区別が困難になる。原理的な検出限界を測っておく必要がある。

## 10. 実行報告

### 10.1 実行報告（2026-07-19 実施）：PoC-M1 — 多価データの検出と場の自己分割（全 PASS）

実装は `tmp/multivalued_poc.py`、結果は `tmp/out/multivalued_poc/`。設定：$H=32$ × 隠れ 2 層、$T=64$、マルチスケール L1 初期化、epochs 600、3 seeds。データは $x\in[-2\pi,2\pi]$ の 128 点に対し $y=\sin x$ と $y=-\sin x$ を各 50%（同じ $x$ に 2 つの $y$）。単一場の学習は両分岐を交互提示して行う（各 epoch で両方を 1 step ずつ = 等頻度提示）。

**結果（seed 0、他 2 seed も同傾向で全判定一致）：**

| 量 | 多価データ（単一場） | 対照（単価値 $\sin x$） |
|---|---|---|
| 残差 MSE | **0.4972** | 0.00071 |
| $\lvert$平均予測$\rvert$ | **0.0296** | — |
| 多峰性スコア | **0.487** | 0.056 |
| 多峰性 p 値 | **0.005** | 0.841 |
| 残差 RMS（$P=1$） | 0.7053 | 0.0775 |
| 残差 RMS（$P=32$） | **0.7051** | 0.0242 |

分割後：分岐 MSE **0.00155 / 0.00167**、自前ユニット **20 / 1**、読み出しコサイン類似度 **−0.999**、割当精度（E-step）**0.988**。

**知見：**

1. **単一場は理論どおり条件付き平均へ潰れた（V-a PASS）。** 平均予測の絶対値は 0.0296（$\mathbb E[y|x]=0$）、残差 MSE 0.497 は条件付き分散 $\mathbb E[\sin^2]=0.5$ とほぼ一致する。ユニットは潤沢に空いており（全 64 動員）、epoch を伸ばしても下がらない — 容量不足でも学習可能性の限界でもないことが対照（単価値 $\sin x$ が同一設定で MSE 0.0007）から確認できる。
2. **既約性の判定が内在ノイズと多価性を完全に分離した。** 多価データの残差 RMS は $P=1\to32$ で 0.7053→0.7051 と**まったく減らない**のに対し、対照は 0.0775→0.0242 と $1/\sqrt P$ 的に減る。「出力がばらつく」ことと「データが多価である」ことを、アンサンブルのパス数依存性だけで区別できる — NNN が確率的であるがゆえに必要になった判別であり、同時に forward だけで完結する。
3. **多峰性検定も分離した（V-b PASS）**：スコア 0.487 / p = 0.005 対 対照 0.056 / p = 0.841。密度は交差活性と同型のしきい値掃引（分布フリー）で読んでいる。
4. **分割は決定的に効く（V-c PASS）**：分岐 MSE 0.0016 は単一場の下げ止まり 0.497 の **1/320**。
5. **第二分岐はほぼ無料（V-d PASS）**：自前ユニットは 20 対 **1**、読み出しコサイン **−0.999**。§4 の予測（分岐は同じ部分空間に住み、$\pm$ 分岐は符号反転で表せる）が定量的に的中し、コンソリデーション稿 §12.9.4 の $\sin$/$-\sin$ の実測値（コサイン −1.00、自前 L2 = 0）を再現した。
6. **割当（E-step）は forward のみで 98.8% の精度**。クロス評価行列と同一の演算で、backward も専用の推論機構も要さない。

**多峰性検出器の 3 つのバグと修正（方法論として記録）。** 初回実装は seed 依存の誤判定を出し、原因を追うと検出器側に三つの欠陥があった。いずれも「統計量の帰無分布を考えずに固定閾値で判定した」ことに起因する。

1. **第二の峰の探し方**：片側範囲の**大域最大**を第二の峰としたため、峰から下り坂の隣接点を拾って谷が消えた（窓幅が小さいときだけ偶然機能していた）。標準的な「最深の谷」定式化 $\max_k[1-\mathrm{dens}[k]/\min(\max\mathrm{dens}[:k],\ \max\mathrm{dens}[k:])]$（prefix/suffix 最大で $O(n)$）に置換。
2. **窓幅の絶対指定**：残差の広がりが大きいと窓あたりの標本が少なくなり、帰無側も揺らぎで最大値に達して**検出力が消える**。窓幅を $h=c\cdot\mathrm{std}$（$c=0.35$）と相対化して解決。
3. **裾の外れ値**：格子を $[\min,\max]$ で張ると、孤立した外れ値の手前で密度が**厳密に 0** になり $\mathrm{score}=1-0/\mathrm{peak}=1$ という偽の谷を作る。これが帰無分布を常に 1.0 に張り付かせ、実データが帰無より低いスコアになる逆転を生んでいた。格子を分位点 $[q_{0.02},q_{0.98}]$ で刈り込んで解決。

修正後の検出器の較正（$h=0.35\,\mathrm{std}$、帰無 = 分散を合わせた単峰ガウス 200 回）：

| 分布 | スコア | p 値 | 判定 |
|---|---|---|---|
| 多価（理想潰れ） | 0.524 | **0.005** | 検出 |
| 単価値ガウス($\sigma$=0.08) | 0.222 | 0.313 | 非検出 |
| 単価値ガウス($\sigma$=0.25) | 0.000 | 1.000 | 非検出 |
| 一様分布 | 0.300 | 0.154 | 非検出 |
| 近接 2 峰（$\pm0.2$） | 0.810 | **0.005** | 検出 |
| 近接 2 峰（$\pm0.1$） | 0.156 | 0.522 | **非検出（検出限界）** |

**教訓**：この統計量は格子上の**最大値統計**なので、標本数・窓幅に依存する帰無分布を持ち、固定閾値での判定は原理的に誤りである。コンソリデーション稿 §12.9.4／§12.9.10 で繰り返し現れた「閉ループの許容値はその体制の平衡から較正せよ／最終品質は絶対基準で守れ」と同型の教訓が、検出器の閾値についても成立する。**判定は較正された p 値で行い、最終的な正当化は分割が実際に残差をユニットコスト以上に下げること（§6 の MDL 判定）に委ねるべきである。**

**限界（§9 の予見の実測による更新）。** 近接 2 峰の検出限界は、本設定では分岐間隔 $\pm0.2$（間隔 0.4 = 内在ノイズ幅 $\sigma_{\mathrm{res}}\approx0.08$ の 5 倍）で検出、$\pm0.1$（同 2.5 倍）で非検出だった。§9.4 の予見「分岐が内在ノイズ幅と同程度なら区別困難」が定量化された形である。また本 PoC は分岐数 2 が既知の設定で分割を行っており、**分割の自動的な起動（holds 異常の監視）と分岐数の決定（MDL 判定）は未実装**である — PoC-M2/M3 の課題。

### 10.2 実行報告（2026-07-19 実施）：PoC-M2 — 部分的多価性と場の部分的重なり（全 PASS）

実装は `tmp/multivalued_partial.py`、結果は `tmp/out/multivalued_partial/`。データは

$$
y=\sin x\ \ (x\le 0,\ \text{単価値}),\qquad
y=\sin x \pm 0.6\sin 2x\ \ (x>0,\ \text{2 分岐})
$$

で、分岐は背景 $\sin x$ を共有し多価領域でのみ違う。$H=32$、$T=64$、マルチスケール初期化、epochs 600、$N=512$ 点、3 seeds。

**結果（seed 0；全 seed で判定一致）：**

| 領域 | 残差 MSE（単一場） | 条件付き検出器 | 周辺多峰性検定 |
|---|---|---|---|
| 多価（$x>0$） | **0.1832** | **1.00** | 0.80 |
| 単価値（$x\le0$） | 0.00052 | **0.00** | 0.40（**偽陽性**） |
| 対照（全域単価値） | 0.00075 | — | — |

分割後：分岐 MSE（全域 / 多価領域）= 0.0020, 0.0071 / 0.0028, 0.0084、自前ユニット **20 / 12**、読み出しコサイン **+0.082**、多価領域での割当精度 **0.988**。

**知見：**

1. **潰れは局所的に起こる（V-a PASS）。** 単一場の残差は多価領域で 0.183（理論床 $\mathrm{amp}^2/4=0.09$ の同オーダー）、単価値領域では 0.00052 と対照（0.00075）と同水準 — **共通部分は単一場でも正しく学べており、多価領域だけが潰れる**。M1（全域で潰れる）との質的な違いが明確に出た。
2. **周辺多峰性検定は局所化に耐えない — 構造的な偽陽性（重要な負の結果）。** 窓ごとの残差の**周辺分布**を見る検定は、多価窓 0.80 に対し**単価値窓でも 0.40 が有意**（$p=0.005$）と出た。原因は方法の側にある：窓内の周辺分布は「$x$ をまたぐ変動」と「同じ $x$ での変動」を混ぜてしまい、単価値領域でも系統的なフィット誤差が $x$ に対して滑らかに振動していれば、その周辺分布はアークサイン型 = 多峰になる（振幅 0.023 の微小な誤差でも形は多峰）。**多価性を意味するのは後者だけなので、$x$ で条件付けない検定は原理的に誤る。** M1（大域・全域で分岐）では偶然この欠陥が露呈しなかった。
3. **条件付き検出器が正しい定式化（V-b PASS）。** 同じ $x$ で観測された $y$ の集合の広がり $\mathrm{spread}(x)$ を、その点での**ネットワーク自身の出力ゆらぎ** $\sigma_{\mathrm{out}}(x)$（単発 forward の std）で割った比を見る：

   $$
   \mathrm{ratio}(x)=\frac{\mathrm{spread}(x)}{\sigma_{\mathrm{out}}(x)},\qquad
   \text{多価} \iff \mathrm{ratio}\gg 1 .
   $$

   これは多価領域 **1.00** / 単価値領域 **0.00** と完全に分離した。分岐ラベルを使わず、$x$ をまたぐ変動を混ぜず、しかも**物差しがネットワーク自身の確率性**である点が NNN 的である（「そもそも区別できない揺らぎの大きさ」を系が自分で提供する）。既約性判定（$P$ 依存性、§10.1）の局所版にあたる。
4. **場は部分的に重なり、コストは多価領域の広さに比例した（V-c PASS）。** 第二分岐の自前ユニットは 12（第一分岐 20 の 6 割）で、M1 の 1 ユニットより明確に高い — 多価領域が入力域の半分を占め、そこでは独立な関数（$\mp0.6\sin2x$）を新たに表現する必要があるためで、予測どおりの振る舞いである。分岐 MSE は多価領域で 0.0028–0.0084 と、単一場の下げ止まり 0.183 の **1/20–1/65**。
5. **読み出しの関係が M1 と質的に違う。** M1（$\pm\sin$、全域で反転）ではコサイン **−0.999** だったのに対し、ここでは **+0.082** — 背景を共有し多価領域でのみ違うため、読み出しは反相関でも同相関でもない中間的な関係になる。**読み出しの類似度が分岐の構造（全域反転か、部分的差異か）を反映する**ことが示された。
6. **割当は多価領域で 98.8%（V-d PASS）。** 単価値領域はどちらの場に入れても等価（don't-care）なので、割当精度は多価領域に限って評価するのが正しい — 部分的多価性では評価指標自体も局所化が要る。

**方法論的な副産物：局所検定の標本密度要件。** 局所多峰性検定は窓あたりの標本数で検出力が決まる。理想残差での事前検証では、$N=128$（窓あたり 25 点）で検出率 **0%**、$N=512$（同 102 点）で **100%** と劇的に変わった。大域検定（M1）は 128 点で足りていたので、**局所化は標本密度の要求を跳ね上げる**。条件付き検出器はこの制約が緩い（各 $x$ での広がりを直接見るため）点でも優れている。

**一般化の際の注意。** 条件付き検出器は「同じ $x$ に複数の $y$ がある」ことを利用する。実データで $x$ が厳密に重複しない場合は $x$ をビン分けする必要があり、そのビン幅は「**ビン内での関数の変化 < 分岐の間隔**」を満たさねばならない（§9.4 の検出限界と同型の分解能条件）。ビン幅がこれを破ると、知見2 と同じ「$x$ をまたぐ変動の混入」が復活する。

### 10.3 実行報告（2026-07-19 実施）：PoC-M3 — 分岐数未知・非等頻度からの場の自己増殖ループ（全 PASS）

実装は `tmp/multivalued_split_loop.py`、結果は `tmp/out/multivalued_split_loop/`。M1/M2 は分岐数 2 を既知として分割した。本 PoC は**分岐数を知らせず**、場を 1 つずつ生やすループでデータに個数を決めさせる。

**データ。** 交差しない 3 分岐（分岐同一性の非識別性 — 交差点で分岐の続きが原理的に曖昧になる問題 — を本 PoC では切り離すため）：$b_0=\sin x$、$b_1=1.5+0.5\sin 2x$、$b_2=-1.8-0.5\sin x$、最小分離 0.3。頻度は main = 5:3:2、rare = 6:3:**1**（10% の稀分岐）。$H=32$、$T=64$、マルチスケール初期化、3 seeds。

**ループ（peel 型；§5 の実装形）。** 未説明率が閾値以上のあいだ：(i) 新しい場を生やす — 学習対象は**未説明点のみ**。warm 期は各 $x$ の未説明候補の**重み付き平均**を目標にし（多数派分岐へ寄る保守的バイアス = §9.1 への対処。平均に一様に最近傍なのが多数派分岐であることは事前に解析で確認）、その後 **WTA 期**：各 $x$ で現在の予測に最も近い未説明候補を目標に採用（サンプルレベルの hard-EM、5 epoch ごと更新）。(ii) WTA 確定後、目標を固定して**精練**（hard-EM の M-step、400 epochs）。(iii) 既存場を凍結語彙とし二層共有（§12.9.9 の share_l1）で anneal-until-stop。(iv) **受理判定（MDL）**：新規説明点（$|r|<0.14$；分岐最小分離 0.3 の半分未満）の重み付き割合が 5% 未満なら場を棄却（領域を掃除）して停止。

**結果（seed 0；3 seeds 全一致で全 PASS）：**

| 設定 | 場の数 | 停止時未説明率 | 分岐 MSE | 自前ユニット | 純度 | 割当精度 |
|---|---|---|---|---|---|---|
| main (5:3:2) | **3** | 0.009 | 0.0014, 0.0038, 0.0012 | 22, 11, **0** | 1.0, 1.0, 1.0 | 1.000 |
| rare (6:3:1) | **3** | 0.037 | 0.0013, 0.0108, 0.0004 | 29, 5, **0** | 1.0, 1.0, 1.0 | 0.995 |

**知見：**

1. **場の個数はデータが決めた（V-a PASS）。** 両設定・全 seed で場はちょうど 3 つ生え、4 つ目の試行は MDL 棄却（ゲイン < 5%）で掃除されて停止した。ループの停止は設計者の分岐数指定ではなく「未説明点の枯渇」と「ゲインの不足」だけで駆動される。
2. **peel 順は頻度順、純度は完全（V-b PASS）。** 場は多数派から順に $b_0\to b_1\to b_2$ を拾い、全場の純度（支配分岐が新規説明点に占める割合）が 1.00 — warm 期の「平均に最近傍の候補へ吸着する」バイアスが、§9.1 で予見したパッチワーク（1 つの場が複数分岐を継ぎ接ぐ）を実際に防いだ。
3. **稀分岐は埋もれない（V-c PASS）。** 10% の $b_2$ も 3/3 seeds で自場を獲得（MSE 0.0004–0.0009）。peel 型では検出が「残差」で駆動されるため、条件付き平均に埋もれる古典的な失敗（頻度で駆動される検出の弱点）を構造的に回避する — ただし本設定の稀分岐は $x$ 全域をカバーしており、被覆まで疎な場合の限界は未測定。
4. **再利用の構造が容量研究の法則どおり現れた（V-d PASS）。** $b_2=-1.8-0.5\sin x$ は $b_0$ の張る空間内 → その場の自前ユニットは **0**（読み出し係数と bias だけで格納）。$b_1$ は $\sin 2x$ という新部分空間 → 自前 5–11。「部分空間の初回だけ全額、以後ほぼ無料」（コンソリデーション稿 §12.9.8）が、教師なしの分岐発見の文脈でもそのまま成立する。
5. **反復の記録：初版の失敗はループでなくフィット品質だった。** 初版（読み出し共有のみ・精練なし・$\varepsilon=0.12$）は純度 1.00・稀分岐回収 3/3 とループの構造は完璧だったが、$b_1$ の場が RMS 0.1–0.2 で頭打ちになり、seed により「4 つ目の拭き取り場が生える」（場 4 つ）か「未説明 14–20% で停止」に分かれた。根本原因は二つ：(i) 場 1 が保守的停止則で 19–25 ユニットを抱え込み細スケール基底が枯渇（コンソリデーション稿 §12.9.8 と同じ既知問題）、(ii) WTA の目標が 5 epoch ごとに揺れて学習予算を浪費。対処はどちらも確立済みの部品 — **share_l1**（後続の場が抱え込まれた L1 基底を読める）と**固定目標の精練フェーズ**（M-step を正しく行う）— で、適用後は全判定が PASS に転じ、実行時間もむしろ短縮された（収束が速くなったため）。
6. **paired 統計の教訓の再確認。** 「拭き取り場が生えるか、未説明が残るか」という初版の二態は、どちらも同じ根本原因（フィット品質）の別の現れだった。ループの正しさとフィットの質は独立に評価すべきで、純度・支配分岐・MDL 棄却の作動はループ側の、分岐 MSE と未説明率はフィット側の指標として分離して読む。

### 10.4 実行報告（2026-07-19 実施）：PoC-M4 — 検出と分割ループのオンライン統合（判定は全 seed 正解；seed 0 の既知上流問題は stop 則修正後の再検証で解消 — 末尾の追記参照）

実装は `tmp/multivalued_online.py`（`split_loop` は past/free を引き継げるよう後方互換リファクタ）、結果は `tmp/out/multivalued_online/`。残課題 2 への対処：M1/M2（検出）と M3（分割）を一つのパイプラインに統合し、**多価かどうかを知らされないタスク列**を一気通貫で処理する。

**タスク列（システムには関数形も分岐数も未知）**：t1 = $\sin x$（単価値）→ t2 = **多価**（$c_0=1.5+0.5\sin 2x$ [w=3]、$c_1=-1.8-0.5\sin x$ [w=2]）→ t3 = $0.7\sin x - 0.5$（単価値・t1 の張る空間内）→ t4 = $\sin 8x$（**学習不能な単価値** = トリガの特異度を試す陰性対照。正規化入力での半ローブ幅 0.125 が最細バンプ幅 0.125 と同じで解像限界）。

**統一パイプライン**：各タスクをまず「条件付き平均への単一場」として素朴に学習（多価を知らないシステムの自然な振る舞い）→ **品質ゲート**（全データ点への重み付きアンサンブル MSE > 0.05）→ 発火時は**三者鑑別**（§3 の表の実装；全て forward 統計）：(i) 容量 = 試行前の空き率 ≥ 25%、(ii) 既約性 = 残差 RMS の $P{=}32/P{=}1$ 比 > 0.8、(iii) 条件付き広がり = $\mathrm{spread}(x)/\sigma_{\mathrm{out}}(x)>1$ の $x$ が 10% 超 → 多価なら素朴な場を掃除して M3 ループへ（past/free を引き継ぐ）、多価でなければ分割せず棄却。

**結果（3 seeds）：**

| タスク | 素朴な単一場の MSE | 既約性 | 広がり | 判定 | 期待 |
|---|---|---|---|---|---|
| t1 $\sin x$ | 0.0013 | — | — | **store** | store ✓ |
| t2 多価 | 2.68（潰れ） | 1.00 | 1.00 | **split** | split ✓ |
| t3 $0.7\sin x{-}0.5$ | 0.0007 | — | — | **store**（自前 0） | store ✓ |
| t4 $\sin 8x$ | 0.50 | 1.00 | **0.00** | **reject** | reject ✓ |

**知見：**

1. **判定は全 seed × 全タスクで正解（V-b 3/3 PASS）。** 特に t4 の鑑別が本 PoC の核心である：既約性は 1.00（多価と同じ）だが**条件付き広がりが 0.00** — 「単一場では説明できない」と「データが多価である」を、同じ forward 統計の 2 軸が分離した。§3 の三者鑑別表が、対照実験なしのオンライン判定として機能することの実証である。
2. **正常経路は透過（V-a、V-d）。** t1/t3 はトリガを踏まず通常格納され、t3 は**自前 0 ユニット**（t2 の分割をまたいで語彙再利用が機能）。全格納記述子の drift は全 seed で厳密ゼロ、格納時と終了時の MSE も一致 — 分割ループの挿入が前後のタスクを一切壊さない。
3. **seed 0 のみ V-a/V-c FAIL — 原因は既知の上流問題（stop 則の保守性）。** t1 の場が **31 ユニット**を抱え込み（保守的停止則のseed 揺らぎの最悪ケース）、t2 の $c_0$（$\sin 2x$）の基底が枯渇 → field 1 のフィット裾が $\varepsilon=0.14$ を超える点を残し、**3 つ目の「拭き取り場」**（純度 1.0・説明率 100% だが、担当分岐の大域 MSE 0.158）が受理された。M3 初版（§10.3 知見5）と同根であり、上流原因はコンソリデーション稿の残課題「stop 則の保守性」（本稿の次の作業対象）そのもの。**stop 則の修正後に本 PoC を再実行して seed 0 が解消することを確認する**のが検証計画である（→ 下の追記のとおり実行し、解消を確認した）。
4. パイプラインの含意：素朴な試行 → ゲート → 鑑別 → 分割/棄却、の全段が既存部品（grow_field・掃除・M3 ループ・条件付き検出器）の**呼び出し順の変更だけ**で構成でき、新しい機構を要しなかった。「多価性への対応」は特別なモードではなく、逐次学習パイプラインの標準経路の一分岐である。

**追記（2026-07-19、stop 則修正後の再検証）**：コンソリデーション稿 §12.9.14 の stop 則 v2 — 層を閉じるまでに複数候補の失敗を許す ban 方式（`stop_layer_fails=4`）と、閉ループ飽和 3 連続でのアニール中断（`stop_abort_saturated=3`）— を有効化して本 PoC を同一条件で再実行した（`tmp/multivalued_online.py --stop-layer-fails 4 --stop-abort 3`、結果は `tmp/out/multivalued_online_v2/`）。**3 seeds × 4 検証項目すべて PASS**。seed 0 では t1 の占有が 31 → **20** ユニットに下がって $\sin 2x$ の細スケール基底が残り、t2 の剥離は場 2 つ（自前 [7,1]、純度 1.0/1.0、未説明 0.9%）で完了 — 拭き取り場は生えなかった（seed 1/2 も own [7,0]/[5,0]、未説明 1.9%/2.3% で PASS）。判定列（store/split/store/reject）と正常経路の挙動（t3 自前 0、全 drift = 0.00e+00）は旧則とまったく同じで、変わったのは領域サイズだけである。検出・分割のロジックに一切触れず、上流の資源問題の修正だけで FAIL が解消したことは、知見 3 の因果診断（stop 則の保守性 → 基底枯渇 → フィット裾 → 拭き取り場）が正しかったことの直接の検証になっている。

**M1–M3 の総括。** 本稿 §1 の構想 — 多価データの検出（M1：既約性 + 多峰性、M2：条件付き広がり比）、場の分割と格納（M1/M2：読み出し共有・部分的重なり）、そして**個数の自動決定**（M3：peel ループ + MDL 棄却）— が一通り実証された。全工程は forward 統計と既存のコンソリデーション部品（動員ダイヤル・凍結語彙・share_l1・anneal-until-stop・掃除）だけで動き、backward・専用の混合モデル機構・分岐数の事前指定をいずれも要しない。残る主要課題：(a) 分岐が交差するデータ（分岐同一性の非識別性と、ネットワークの平滑性が与える暗黙の接続事前分布）、(b) 検出（M1/M2）と分割ループ（M3）の**オンライン統合** — holds 異常を引き金に自動で M3 ループへ入る枠組み、(c) $x$ が厳密に重複しない実データでのビン分け（§10.2 の分解能条件）、(d) 稀分岐の被覆まで疎な場合の検出限界。
